# Dev Switches (Stage A3)

用途：缩短开发回归路径，不替代正式玩法验证。

## 键位与按钮
- `F8` / DevTools: `开启/关闭极速战斗(F8)`
  - 行为：切换战斗速度倍率（x4 ↔ x16）。
  - 安全边界：仅影响回合推进速度，不修改战斗胜负计算。

- `F9` / DevTools: `快速开局(F9)`
  - 行为：从地图态快速推进到可操作的准备阶段；会自动处理中间的奖励/海克斯态。
  - 安全边界：不直接发放额外资源，仅做流程推进。

- `F10` / DevTools: `跳关胜利(F10)`
  - 行为：当前层强制胜利并自动处理奖励/海克斯，直到楼层推进。
  - 安全边界：用于流程联通/后续层验证，不用于平衡结论。

- `F11` / DevTools: `直达Boss(F11)`
  - 行为：自动推进到 Boss 层附近流程入口（Prepare/Battle/Reward/Hex）。
  - 安全边界：用于后期节点与 Boss 相关链路压测，不用于真实通关难度评估。

## 使用建议
1. 流程稳定性回归：优先 `F9 -> F10` 组合。
2. 后期链路验证：直接 `F11`，观察 Boss 前后流转是否正常。
3. 平衡验证：不要依赖 F10/F11 结果，需走真实战斗或批量平衡回归。

## 与回归门禁关系
- UI 烟雾回归已覆盖：
  - 快速开局可进入准备
  - 跳关可推进楼层
  - 跳Boss可到后期流程
- Batch 总门禁仍以 `DevRunRegression3FloorsBatch()` 为准。
